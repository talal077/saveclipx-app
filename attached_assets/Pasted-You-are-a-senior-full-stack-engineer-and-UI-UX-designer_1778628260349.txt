BACKEND REQUIREMENTS
==================================================

Create two main endpoints:

1. POST /api/x-video

Purpose:
Extract video information and available qualities from a public X/Twitter post URL.

Request:
{
  "url": "https://x.com/user/status/123"
}

Behavior:
- Validate URL
- Accept only x.com and twitter.com
- Use yt-dlp-exec to extract video information
- Extract available MP4 formats
- Remove duplicate qualities
- Sort qualities from highest to lowest
- Return clean JSON

Response:
{
  "success": true,
  "title": "Video title",
  "thumbnail": "thumbnail url",
  "duration": 120,
  "formats": [
    {
      "quality": "720p",
      "url": "direct video url",
      "ext": "mp4",
      "filesize": 12345678
    }
  ]
}

2. GET /api/download-file

Purpose:
Force direct file download through the backend.

Query params:
- url = encoded direct video URL
- filename = desired filename

Critical backend behavior:
- Fetch the video URL from the server side
- Stream the response to the user
- Set headers:
  Content-Type: video/mp4
  Content-Disposition: attachment; filename="x-video.mp4"
- This endpoint must force download instead of opening the video in the browser
- Do not redirect to the raw video URL
- Do not send the user to video.twimg.com

Example backend behavior:

app.get("/api/download-file", async (req, res) => {
  const videoUrl = req.query.url;
  const filename = req.query.filename || "x-video.mp4";

  const response = await fetch(videoUrl, {
    headers: {
      "User-Agent": "Mozilla/5.0",
      "Referer": "https://x.com/"
    }
  });

  res.setHeader("Content-Type", "video/mp4");
  res.setHeader("Content-Disposition", `attachment; filename="${filename}"`);

  response.body.pipe(res);
});
