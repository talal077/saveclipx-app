3. POST /api/download

Purpose:
Download the selected video server-side and send the final MP4 file to the browser.

Request body:
{
  "url": "https://x.com/user/status/123",
  "formatId": "selected-format-id"
}

Critical behavior:
- Use yt-dlp-exec to download the selected format.
- Use ffmpeg-static.
- Force MP4 output.
- If the selected format is HLS/m3u8, yt-dlp must download the segments and ffmpeg must merge them into a real MP4.
- Save temporary output into /tmp/x-downloads.
- Use unique filename with crypto.randomUUID().
- Stream the final .mp4 file to the response.
- Set headers:
  Content-Type: application/octet-stream
  Content-Disposition: attachment; filename="x-video.mp4"
- Delete the temporary MP4 file after response finishes.
- Do not redirect.
- Do not send the raw URL.
- Do not return m3u8.
- Do not open video.twimg.com.
- If MP4 creation fails, return a clean JSON error.

yt-dlp options must include equivalent of:
- format: selected format + bestaudio fallback when needed
- mergeOutputFormat: mp4
- recodeVideo: mp4 if needed
- ffmpegLocation: ffmpeg-static path
- output: /tmp/x-downloads/unique-id.%(ext)s
- noPlaylist: true
- noWarnings: true
- addHeader:
  - referer:https://x.com/
  - user-agent:Mozilla/5.0

Important:
The final response of /api/download must be a binary MP4 file, not JSON on success.
